# Mesh — Marketing Website Improvement Plan (meshsports.co)
_Review captured 2026-07-21. The app is at app.meshsports.co; this doc is about the public
marketing site at meshsports.co. Parked for a dedicated pass later._

## Overall read
The bones are solid — a conventional, well-ordered SaaS landing page:
**hero → problem → features → how it works → pricing → FAQ → CTA.** The problem section is
genuinely good, and "Stop juggling. Start coaching." is a strong, ownable hero line. The gaps
are **content accuracy, missing proof, and a lack of visuals** — not layout.

---

## Priority to-do (ranked)

2. **Fix the Playbook feature** — remove it or mark "Coming soon."
3. **Confirm the beta modal actually captures an email.**

5. Add **one real testimonial** or a short **founder line**.
6. Polish: **emoji → SVG icons**; pull the Hudl/TeamSnap contrast up into the hero.

## The three biggest fixes

### 1. Zero product screenshots (highest impact)
The only image on the whole page is the logo — everything else is text + emoji. Mesh is a
visual product (practice grid, depth chart, calendar, roster). Visitors currently have to take
it on faith. One clean phone mockup in the hero + screenshots in a few feature cards will move
conversion more than any copy change.

### 2. Playbook is marketed but was parked
"Playbook — Play diagrams, Formations, Staff access" is featured as a core capability, but in
the app it's a "coming soon" screen (parked in v40.16). That's a promise the product doesn't
keep. **Remove it from the features grid, or explicitly label it "Coming soon."**

### 3. Verify the beta CTA captures a lead
On opening the "Request beta access" modal, no email input was visible — only the heading,
"We'll review your request and send you a code within 24 hours," and a close button. If that
modal doesn't collect an email, the primary conversion action is leaking every lead. **Verify.**
Minor: the three "Request beta access" buttons open the modal, but the final one links straight
to app.meshsports.co instead — align them.

---

## Missing content
- **Safety / witness messaging.** Auto-witnessed player DMs are a real differentiator
  (SafeSport-style protection) and a trust-builder for parents/ADs — completely absent. Make it
  a feature, maybe a small "built for youth-sports safety" trust section.
- **Social proof.** The closing CTA says "Join the coaches already using Mesh," but there's no
  quote/logo/number. Even one real beta-coach sentence beats an implied crowd. If none yet,
  soften that line so it doesn't over-promise.
- **"Who's behind this."** Claims "built by coaches, for coaches" but never says by whom. A
  two-sentence founder note buys trust for an unknown product handling kids' data.
- **Mobile / works-on-any-phone.** Add-to-home-screen + push notifications with the app closed.
  Coaches care about "does this work on my phone."
- **Data privacy for minors.** Generic "is my data secure?" FAQ exists, but lean into
  COPPA/SafeSport-minded, role-based access (players see only what they should).

## Trim / reconsider
- **Emoji feature icons** read as placeholder — swap to a consistent SVG set (cheap, high impact).
- **"0 spreadsheets needed"** stat is fine but generic; replace with real numbers once available
  (programs onboarded, hours saved).
- **"Any sport, any level" vs. football-coded language** (position groups, scripts, depth chart,
  pads). Be intentional: lean into football as the wedge in examples/imagery while keeping "any
  team sport" as the expansion story. Sport-agnostic everywhere reads as less made-for-me.

## Copy & positioning
- Copy quality is good — punchy, benefit-led. Keep it.
- The Hudl/TeamSnap FAQ handles the right objection but it's buried. Surface the contrast near
  the hero: **TeamSnap does logistics, Hudl does film — Mesh runs the program.**
- If the site is heavily ALL-CAPS, keep caps for headers but not body paragraphs (readability).

---

## Structure notes (what's already working — don't touch)
- Problem section with three concrete pain points.
- Tabbed features (Coaching / Program / Communication / Operations) — avoids a wall of features.
- Clear "Up and running in under 10 minutes" 3-step setup.
- Honest beta pricing ("free during beta, grandfathered in").
- FAQ covers the right questions (sports/levels, vs competitors, parent/player access, security,
  access).
